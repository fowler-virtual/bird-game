# 鳥アニメーションの画像レイアウト比較

Common の 4 コマをどう用意するかで、実装のしやすさが変わります。

---

## 1. 4 枚の別画像（いちばんシンプル）

**例**: `common-1.png`, `common-2.png`, `common-3.png`, `common-4.png`

| メリット | デメリット |
|----------|------------|
| 実装が簡単（img の `src` を切り替えるだけ） | ファイルが 4 つになる |
| background-position 不要 | HTTP リクエストが 4 本（キャッシュすれば 2 回目以降は軽い） |
| レイアウトやバグの原因が少ない | 画像の差し替え時に 4 枚そろえる必要がある |
| どの環境でも動きが分かりやすい | |

**実装イメージ**:  
1 つの `<img>` を置き、 setInterval や CSS アニメーションで 2 秒ごとに `src` を `common-1.png` → `common-2.png` → … → `common-1.png` に変える。

---

## 2. 横一列のスプライトシート（1 ファイルで済む）

**例**: 1 枚の PNG に [コマ1 | コマ2 | コマ3 | コマ4] を横に並べる（例: 500×500 を 4 枚なら 2000×500）

| メリット | デメリット |
|----------|------------|
| 画像ファイルは 1 つでよい | 2×2 より「ずれ」の計算が楽（x だけ 0%, 33.33%, 66.66%, 100%） |
| 1 要素で `background-position` の x だけを steps で切り替えればよい | 横長の画像になる |
| 縦方向を動かさないので、スライドして見えにくい | 既存の 2×2 とはレイアウトが違うので画像の作り直しが必要 |

**実装イメージ**:  
1 つの div に `background-size: 400% 100%`、  
`background-position` を `0 0` → `33.33% 0` → `66.66% 0` → `100% 0` → `0 0` と `steps(4)` で切り替える。  
（2×2 のときのような「横スライド」は、x だけの切り替えなので出にくい。）

---

## 3. 2×2 のスプライトシート（いまの方式）

**例**: 1 枚の PNG で [左上・右上 / 左下・右下] の 4 コマ

| メリット | デメリット |
|----------|------------|
| 1 ファイルで済む | background-position が x,y 両方になり、スライドして見えやすい |
| 既存の common.png をそのまま使える | そのため 4 レイヤー＋opacity で切り替える必要があり、実装が重い |

---

## おすすめ

- **実装の簡単さ優先**: **4 枚に分ける**（common-1.png ～ common-4.png）。  
  表示は 1 枚の `<img>` で、2 秒ごとに `src` を変えるだけでよい。

- **ファイル数を 1 枚にしたい**: **横一列のスプライト**。  
  画像を [コマ1][コマ2][コマ3][コマ4] の 1 行に並べ、`background-position` の x だけを 0% → 33.33% → 66.66% → 100% で steps 切り替え。

いまの 2×2 のままでも動くが、**4 枚に分けるか、横一列にすると実装もデザインも扱いやすくなります**。
