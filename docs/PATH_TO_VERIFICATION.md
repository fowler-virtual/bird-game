# ①にたどり着くための道筋

> **① 達成済み（2026-02-27）**: PC 環境とスマホ環境（MetaMask ブラウザ）で Claim・ガチャ・デッキ編成を交互に実施し、問題なく動作することを確認。以下のチェックリスト 1〜9 を両環境でクリア。

**目標**: PC 環境・スマホ環境（MetaMask / Rabby ウォレット内ブラウザ）で不具合なく一通り動くことを確認し、その後にゲームバランス調整（②）に進める状態にする。

---

## ①「一通り動く」の定義（確認用チェックリスト）

以下を **PC（Chrome）** と **スマホ（MetaMask または Rabby 内ブラウザ）** の両方で、**デプロイ版（Git または Vercel）** を使って確認する。

| # | 項目 | 確認内容 | 状態 |
|---|------|----------|------|
| 1 | タイトル表示 | 起動して Connect Wallet が表示され、白画面・崩れがない | 確認済み |
| 2 | 接続（1回目） | Connect を押すとウォレットが開き「接続」でゲーム画面に遷移する | 確認済み |
| 3 | 接続（2回目・SIWE） | 続けて署名用のウォレットが開き、署名で完了する（1クリックで接続→署名の2回まで） | 確認済み |
| 4 | ゲーム表示 | タブ（FARMING / ADOPT / LOFT / NETWORK）が表示され、SEED・$SEED 等が表示される | 確認済み |
| 5 | Sepolia 切り替え | 接続後に Sepolia に切り替わる（または追加して切り替え） | 確認済み |
| 6 | ガチャ | ADOPT でガチャが実行でき、鳥がインベントリに増える | 確認済み |
| 7 | デッキ配置 | LOFT で鳥をスロットに配置できる | 確認済み |
| 8 | SAVE | LOFT の SAVE で「保存: 成功」となり、再読み込みしても状態が保持される | 確認済み |
| 9 | Claim（任意） | Claim ボタンが押せ、ウォレットが開く。on-chain で成功する | 確認済み |

**① 完了の目安**: 1〜8 が両環境（PC・スマホ）で問題なくできること。9 は「ボタンが反応しウォレットが開く」まででよく、オンチェーン成功は環境整備後に確認してもよい。

## ①で解消されたブロッカー

### ブロッカー A: 2回目ウォレット（SIWE 署名）が開かない — 解決済み（2026-02-24）

- 処理順序変更、pending nonce 対応、nonce フォールバック、VITE_CLAIM_API_URL の Vercel 設定等により解消。

### ブロッカー B: Claim がオンチェーンで revert する — 解決済み（2026-02-24）

- 新しい `RewardClaim.sol` を Sepolia に再デプロイ。pool の approve 実施、Vercel 環境変数更新。

## こちら側でできる動作確認（自動・ローカル実行）

以下はブラウザやウォレットを開かずに実行できるため、開発側（AI や CI）でも実施できる。

| 確認内容 | やり方 |
|----------|--------|
| **ビルド** | `npm run build` で TypeScript と Vite ビルドが通るか |
| **デプロイ先の疎通** | `npm run verify:deployed` または `node scripts/verify-deployed.mjs [URL]` |
|  | トップが 200 で返るか、HTML に「Bird Game」または「Connect」が含まれるか |
|  | GET `/api/auth/nonce` が 200 で JSON に `nonce` が返るか |
|  | GET `/api/claim/signer` の応答（200 なら signer 設定済み） |

**できないこと**: ブラウザで Connect を押す・ウォレットのポップアップを操作する・SIWE 署名から SAVE までの一連の流れを実際に実行する。これらは人手またはブラウザ自動化（E2E）が必要。
