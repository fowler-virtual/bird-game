# スマホで game-state の GET が 200 か確認する方法

スマホブラウザから「GET /game-state が 200 になっているか」を確認する主な方法です。

---

## 方法 1: Android + Chrome のリモートデバッグ（おすすめ）

1. **スマホ側**
   - 設定 → **開発者向けオプション** を有効にする。
   - **USB デバッグ** をオンにする。
   - USB で PC と接続し、接続時に「USB デバッグを許可」する。

2. **PC 側（Chrome）**
   - スマホでゲームを開き、**Connect** してゲーム画面まで進めておく。
   - PC の Chrome で `chrome://inspect` を開く。
   - **Remote Target** にスマホのページが表示されるので、該当ページの **inspect** をクリック。

3. **DevTools が開いたら**
   - **Network** タブを開く。
   - スマホでゲームを **リロード** するか、一度 Disconnect して **もう一度 Connect** する。
   - フィルタに `game-state` と入力する。
   - **game-state** のリクエストをクリック → **Headers** の **Status Code** が **200** か **401** か確認する。

---

## 方法 2: iPhone + Safari の Web インスペクタ

1. **iPhone 側**
   - 設定 → **Safari** → **詳細** → **Web インスペクタ** をオンにする。

2. **Mac 側（Safari）**
   - Safari の **環境設定** → **詳細** → **メニューバーに「開発」メニューを表示** にチェック。
   - iPhone を USB で接続し、スマホの Safari でゲームを開く。
   - Mac の Safari の **開発** メニュー → **[iPhone 名]** → 該当ページを選ぶ。

3. **開発者ツールが開いたら**
   - **ネットワーク** タブで、リロードまたは再接続後に **game-state** を探し、ステータスが 200 か確認する。

---

## 方法 3: モバイル用コンソール（Eruda）を一時的に入れる

コードに一時的に「Eruda」などのモバイル用 DevTools を追加すると、スマホの画面上でコンソールやネットワークを確認できます。

- 本番に長く入れない場合は、**開発時だけ** 使う想定で検索キーワード: `eruda` や `vconsole`。

---

## 確認のポイント

- **200**: サーバーから状態を取得できている。PC で保存した進行が Redis にあれば、その内容が返っている。
- **401**: 認証されていない（Cookie が送られていない、または SIWE 未完了）。この場合は初期状態になる。
